\section{Methodology}

The gPCD framework integrates broad-phase occupancy generation directly
into the graphics pipeline while preserving the conventional
broad-phase/narrow-phase organization of collision detection.
Figure~\ref{fig:overview_architecture}(a) illustrates the conventional
organization in which rendering and collision detection operate as
separate processes in many GPU-based collision-detection frameworks.
Figure~\ref{fig:overview_architecture}(b) shows the gPCD framework,
where broad-phase occupancy generation is integrated directly into the
graphics pipeline, allowing occupancy construction and rendering to
proceed concurrently.

Following framework initialization, collision detection proceeds through
a broad-phase occupancy-generation stage and a narrow-phase evaluation
stage. Occupancy information is first constructed within the graphics
pipeline and subsequently used to identify candidate interactions for
narrow-phase evaluation..

\subsection{Broad-Phase Occupancy Generation}

Broad-phase occupancy generation is implemented within the vertex stage
of the graphics pipeline. Each particle is represented by an
axis-aligned bounding box (AABB) and independently evaluates the cell
locations intersected by its eight AABB corners. The resulting cell
occupancy information is stored within the cell-array structure and used
during narrow-phase candidate discovery.

Let $(x,y,z)$ denote the floating-point particle position and let
$r$ denote the particle radius. The eight corner locations are
constructed as

\begin{equation}
\mathbf{x}_c =\mathbf{x}+(\pm r,\pm r,\pm r),
\end{equation}

where $c \in \{0,\ldots,7\}$ identifies one of the eight AABB corners.
Each corner location is subsequently converted to a cell coordinate and
used to determine particle occupancy within the cell array.

Particle diameters are constrained to be less than or equal to one.
Under this constraint, a particle may intersect at most eight
neighboring cells, allowing occupancy to be determined entirely from
the eight AABB corners.

Each corner location is mapped to a cell coordinate within the uniform
cell array by rounding to the nearest integer cell center,

\begin{equation}
X = \mathrm{round}(x),\qquad Y = \mathrm{round}(y),\qquad Z = \mathrm{round}(z).
\end{equation}

Cells containing a zero-valued coordinate component are reserved to
form a guard layer surrounding the active domain. Consequently, valid
occupancy cells satisfy

\begin{equation}
X>0,\qquad Y>0,\qquad Z>0.
\end{equation}

Cell coordinates are then converted into a flattened cell index,

\begin{equation}
I = X + W(Y + HZ).
\end{equation}

The flattened index serves as the primary cell identifier throughout
the collision-detection pipeline.

Each particle maintains a \emph{particle corner occupancy list} containing the flattened cell identifiers associated with its eight AABB corners. These cell identifiers are generated during occupancy construction and retained for subsequent narrow-phase evaluation.

During occupancy generation, the cell identifiers stored in the particle
corner occupancy list are used to locate the corresponding \emph{cell-array particle occupancy lists} and insert the particle identifier into each occupied cell.

Because occupancy generation is particle-driven, only cells referenced
by particle AABB corners are accessed during broad-phase construction.
Empty cells are never traversed during occupancy generation. Therefore,
for sparse particle configurations, broad-phase work depends on the
number of particles and their occupied cells rather than the total
number of cells in the cell array.

As illustrated in \Figo{CellSpansA}, multiple AABB corners from
the same particle may map to a common cell. These \emph{duplicate corner] references are suppressed during occupancy generation so that only a
single copy of the particle identifier is stored within a given
cell-array particle occupancy list.

Following duplicate suppression, insertion into shared cell-array occupancy lists is performed using an atomic operation on a \emph{slot reservation array} the same length as the cell array. In this way, an atomic slot counter is availble for each cell-array occupancy list to produce a unique slot within the target list.

The resulting cell--occupancy lists constitute the potentially colliding set (PCS) used during narrow-phase evaluation. Consequently, occupancy generation directly constructs the PCS without requiring a separate candidate-construction stage.

\subsection{Narrow-Phase Evaluation}

During narrow-phase evaluation, each source particle traverses its
particle corner occupancy list and accesses the corresponding
cell-array occupancy lists. The particles referenced by these
cell-array occupancy lists constitute the candidate neighbors considered for collision detection. Each candidate is subsequently evaluated to determine contact.

As illustrated in \Figo{CellSpansB}, the same source--target
interaction may be encountered through multiple occupied cells during
occupancy-list traversal. To prevent \emph{duplicate collision} evaluations, each particle maintains a collision list containing the identifiers of previously registered interactions. When a collision is detected, the target particle identifier is added to this list, and subsequent evaluations of the same source--target pair are discarded.

Because candidate discovery is performed independently for each source
particle, collision evaluation remains source-owned throughout the
narrow phase. Consequently, a globally shared collision-pair structure
is not required, reducing synchronization requirements during massively
parallel execution.
