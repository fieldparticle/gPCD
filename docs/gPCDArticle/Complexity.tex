\section{Complexity Analysis}

The computational complexity of gPCD is determined primarily by
broad-phase occupancy generation and narrow-phase collision evaluation.

\subsection{Occupancy Generation}

Each particle performs a fixed number of occupancy evaluations
independent of the total particle count. Let
$c_{\mathrm{occ}}$ denote the bounded occupancy-generation cost per
particle. The resulting broad-phase cost is

\begin{equation}
T_{\mathrm{occ}}(N)=c_{\mathrm{occ}}N=O(N).
\end{equation}

Because occupancy lists are accessed directly through flattened cell
indices, insertion and retrieval remain (O(1)) operations.

\subsection{Narrow-Phase Evaluation}

Each source particle traverses the occupancy lists associated with its
corner cells. Let $K$ denote the total number of interaction
evaluations performed during narrow-phase execution. Since each
interaction requires bounded work,

\begin{equation}
T_{\mathrm{np}}(K)=O(K).
\end{equation}

Consequently, narrow-phase complexity depends on the number of
candidate interactions represented within the potentially colliding
set rather than the total particle count alone.

\subsection{Storage Requirements}

Particle storage scales linearly with particle count,

\begin{equation}
M_{\mathrm{particle}}=O(N).
\end{equation}

For a cubic cell array of side length (S) and occupancy-list width
(W),

\begin{equation}
M_{\mathrm{cell}}=O(S^3W).
\end{equation}

Since (W) is fixed,

\begin{equation}
M_{\mathrm{cell}}=O(S^3).
\end{equation}

The total storage requirement is therefore

\begin{equation}
M_{\mathrm{gPCD}}=O(N+S^3W),
\end{equation}

or $(O(N+S^3))$ for fixed occupancy width.

\subsection{Overall Complexity}

Combining broad-phase occupancy generation and narrow-phase evaluation
gives

\begin{equation}
T_{\mathrm{gPCD}}=O(N+K),
\end{equation}

where \(N\) is the number of particles and \(K\) is the number of
candidate interactions evaluated during narrow-phase execution.

The corresponding storage requirement is

\begin{equation}
M_{\mathrm{gPCD}}=O(N+S^3W),
\end{equation}

which reduces to \(O(N+S^3)\) for fixed occupancy width.

The primary contribution of gPCD is not the introduction of a new
asymptotic complexity class. Similar to other direct-insertion spatial
subdivision approaches, gPCD exhibits linear broad-phase occupancy
generation complexity. Rather, its contribution lies in the
architectural organization of graphics-pipeline occupancy generation,
duplicate-aware cell construction, and source-owned collision
evaluation within a unified GPU-driven framework.
