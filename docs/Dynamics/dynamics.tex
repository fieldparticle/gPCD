\section{Purpose}

The current dynamics model treats particles as soft, purely repulsive disks.
Contact response is derived from overlap geometry and from a force-law estimate
of the zero-velocity point. Walls are represented by ghost particles so that
wall collision response follows the same geometry path as particle-particle
collision response.

The implementation is split between \texttt{Base.py} and
\texttt{NeoDynamics.py}. \texttt{Base.py} owns particle storage, wall/contact
detection, contact state, cluster solving, and application of predicted
velocities. \texttt{NeoDynamics.py} owns the force-law geometry calculations
used to estimate compression, rebound, and the zero-velocity state.

\section{Particle State}

Each active particle has position
\[
    \mathbf{x}_i = (x_i, y_i),
\]
velocity
\[
    \mathbf{v}_i = (v_{xi}, v_{yi}),
\]
mass \(m_i\), and radius \(r_i\). Contacts are detected from center distance.
For particles \(i\) and \(j\),
\[
    d_{ij} = \lVert \mathbf{x}_j - \mathbf{x}_i \rVert.
\]
A particle-particle contact exists when
\[
    d_{ij} < r_i + r_j.
\]

The contact normal used by the source particle points from the source particle
toward the target particle:
\[
    \mathbf{n}_{ij}
    =
    \frac{\mathbf{x}_j - \mathbf{x}_i}{d_{ij}}.
\]
If the centers coincide, the code chooses a stable fallback normal.

\section{Overlap Area}

For the current equal-radius implementation, overlap area is the circular lens
area
\[
    A(d, r)
    =
    2r^2 \cos^{-1}\left(\frac{d}{2r}\right)
    -
    \frac{d}{2}\sqrt{4r^2 - d^2}.
\]
The implementation clamps \(d\) into \([0, 2r]\) before evaluating the formula.

The model uses overlap area as the geometric progress variable. Larger overlap
means deeper compression. During rebound, decreasing overlap means stored
normal momentum is being released.

\section{Force Law And Zero-Velocity Point}

The Neo model estimates the maximum penetration from the force law
\[
    F(\alpha) = Q\alpha^2,
\]
where \(Q\) is configured by \texttt{collision\_stiffness\_q} and
\(\alpha\) is penetration depth.

For two particles, the incoming normal speed is
\[
    v_{\mathrm{in}}
    =
    \max\left(0,\,
    -(\mathbf{v}_j - \mathbf{v}_i)\cdot \mathbf{n}_{ij}
    \right).
\]
The reduced mass is
\[
    \mu = \frac{m_i m_j}{m_i + m_j}.
\]
The estimated zero-velocity penetration is
\[
    \alpha_0
    =
    \left(
        \frac{3\mu v_{\mathrm{in}}^2}{2Q}
    \right)^{1/3}.
\]
The zero-velocity center distance is
\[
    d_0 = r_i + r_j - \alpha_0.
\]
The zero-velocity overlap area is
\[
    A_0 = A(d_0, r).
\]

If the current overlap is already greater than the force-law zero overlap,
the zero state is promoted to the current overlap. This prevents the solver
from trying to rewind a contact past the deepest overlap already observed.

\section{Compression And Rebound}

The zero-velocity point is the turnaround point. Before this point the contact
is in compression. After this point the contact is in rebound.

For a contact with current overlap \(A\), the normalized compression fraction is
\[
    f_c = \mathrm{clamp}\left(\frac{A}{A_0}, 0, 1\right).
\]
The rebound fraction is
\[
    f_r = 1 - f_c.
\]

The compression velocity fraction is
\[
    s_c = \sqrt{1 - f_c},
\]
and the rebound velocity fraction is
\[
    s_r = \sqrt{f_r}.
\]

The incoming normal momentum stored at the zero-velocity point is
\[
    p_0 = \mu v_{\mathrm{in}}.
\]
During compression, the model moves velocities toward the zero-velocity
turnaround velocity. During rebound, the model releases stored normal momentum
according to \(s_r\).

\section{Contact State}

Contact state is required because a collision spans many time steps. The state
stores:

\begin{itemize}[noitemsep]
    \item the target particle or wall id,
    \item the current phase, either compression or rebound,
    \item the first-contact velocities,
    \item the first-contact normal,
    \item the zero-velocity overlap area \(A_0\),
    \item the zero-velocity center distance \(d_0\),
    \item diagnostic internal momentum fields.
\end{itemize}

Particle-particle and wall-contact state are both stored in source-owned
\texttt{gcs} slots. Particle contacts use the target particle index as the
\texttt{target\_index}. Wall contacts use a wall target marker:
\[
    \texttt{("wall", wall\_flag)}.
\]
The current-frame wall list, \texttt{bcs}, still records which walls are active
for detection, reporting, and visualization. The persistent collision history,
however, lives in \texttt{gcs} for both particle and wall contacts.

\section{Wall Ghost Particles}

Walls are modeled as ghost particles. This keeps wall response close to the
particle-particle response instead of using a separate wall force function.

For a particle of radius \(r\), the ghost particle has the same radius and a
large effective mass:
\[
    m_g = 10^{12}m.
\]
The large mass makes the wall effectively fixed while still allowing the same
two-body geometry code to be used.

\subsection{Wall Skin}

The wall ghost is intentionally moved inward by a small skin distance
\[
    s = \gamma r,
\]
where
\[
    \gamma = 0.125.
\]
This means the ghost perimeter lies inside the wall by one eighth of the
particle radius. For \(r = 0.25\), the skin is
\[
    s = 0.03125.
\]

For the four walls, ghost centers are placed as follows:
\[
\begin{aligned}
    x_g &= x_{\min} - r + s && \text{left wall},\\
    x_g &= x_{\max} + r - s && \text{right wall},\\
    y_g &= y_{\min} - r + s && \text{bottom wall},\\
    y_g &= y_{\max} + r - s && \text{top wall}.
\end{aligned}
\]

The contact detector must use the same skin. For example, left-wall contact
begins when
\[
    x - r < x_{\min} + s,
\]
and right-wall contact begins when
\[
    x + r > x_{\max} - s.
\]

This deliberately sacrifices a small amount of box space. The benefit is a
more stable soft-wall interaction with less special-case cleanup code. The wall
is still represented by a ghost particle; the only modeling choice is that the
ghost perimeter is placed slightly inside the visible boundary.

\section{Wall Normal Velocity}

Wall normals point outward from the domain:
\[
\begin{aligned}
    \mathbf{n}_{\mathrm{left}} &= (-1, 0),\\
    \mathbf{n}_{\mathrm{right}} &= (1, 0),\\
    \mathbf{n}_{\mathrm{bottom}} &= (0, -1),\\
    \mathbf{n}_{\mathrm{top}} &= (0, 1).
\end{aligned}
\]
The particle wall-normal velocity is
\[
    v_n = \mathbf{v}\cdot\mathbf{n}.
\]
Positive \(v_n\) means the particle is moving into that wall. Negative \(v_n\)
means the particle is moving away from it.

In the ghost-particle interpretation, the relative normal velocity against a
fixed wall is equivalent to a particle moving against a very massive target.
The implementation uses the wall ghost and fixed-target handling to preserve
the particle tangent velocity while changing only the wall-normal component.

\section{Clusters}

When multiple contacts are active at the same time, \texttt{Base.py} builds
contact clusters. A cluster contains particles connected by active
particle-particle contacts plus any active wall contacts on those particles.

For each contact, the model computes a desired change in relative normal
velocity. The cluster solver forms a linear system whose unknowns are normal
impulses. Applying those impulses gives velocities that satisfy all active
contact predictions for the cluster as consistently as possible.

Wall contacts are represented as contacts with no movable target. Their target
mass is effectively infinite, so only the source particle receives the impulse.

\section{Zero-Overlap Clipping}

After velocity prediction and movement, the code clips rebound contacts that
would penetrate deeper than their stored zero-velocity geometry. For particle
pairs, this moves both particles along the contact normal according to their
masses. For wall contacts, it moves the source particle so that its distance to
the skinned ghost matches the stored zero-center distance.

For a wall contact with target center distance \(d_0\), the distance from the
particle center to the visible wall line is
\[
    d_{\mathrm{wall}} = d_0 - r + s.
\]
This formula must include the skin \(s\). Omitting the skin makes the clipper
place the particle at the wrong effective wall distance and can keep a particle
stuck in wall contact.

\section{Starting Overlaps}

If particles start at \(t=0\) already overlapping, the model treats the contact
as post-turnaround rebound. The same rule applies to starting wall contacts.
The first-contact velocity is reconstructed when needed so the force-law zero
state is still based on an incoming collision.

\section{Current Design Rules}

\begin{enumerate}[noitemsep]
    \item Contacts are geometric overlaps, not independent force proxies.
    \item Walls are ghost particles, not a separate collision model.
    \item The wall skin is part of wall geometry and must be used consistently
          by detection, ghost placement, prediction, clipping, reporting, and
          visualization.
    \item Tangential velocity is preserved by frictionless contacts.
    \item Rebound contacts must not be allowed to move deeper than their stored
          zero-velocity overlap.
    \item Contact state is source-owned wherever possible to keep the model
          compatible with future GPU execution.
\end{enumerate}

\section{Code Map}

\begin{itemize}[noitemsep]
    \item \texttt{Base.step}: runs contact detection, state update, prediction,
          movement, and clipping.
    \item \texttt{Base.detect\_contacts}: rebuilds particle and wall contact
          lists.
    \item \texttt{Base.wall\_ghost\_contact}: constructs skinned wall ghost
          geometry.
    \item \texttt{Base.geo\_velocity\_predictions}: builds active contact
          clusters.
    \item \texttt{Base.geo\_contact\_cluster\_prediction}: solves multi-contact
          particle and wall clusters.
    \item \texttt{Base.clip\_zero\_velocity\_wall\_overlaps}: prevents rebound
          wall contacts from exceeding their zero-velocity geometry.
    \item \texttt{NeoDynamics.particle\_zero\_velocity\_state}: computes the
          force-law zero state for particle-particle contacts.
    \item \texttt{NeoDynamics.starting\_overlap\_report}: computes compression
          and rebound velocity reports for the current overlap.
\end{itemize}
